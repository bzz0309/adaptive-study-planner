# 项目协作说明

## 产品定位

Adaptive Study Planner 是面向 TOPIK 备考的学习行为驱动系统：

计划 -> 完成练习 -> 自动记录 -> 错题与复盘 -> 下一次学习重点。

奖励系统只承担 Day1/Day7 等完成后的轻反馈，不替代学习主线。

- Day1/Day7 触发只消费真实完成记录；Day7 必须是以本次完成日期为结尾的连续7个北京时间自然日，不能用累计打卡天数代替。主站只决定节点，Day1/Day7 展示继续交给冻结的 `reward-companion-system` 场景。

## 开发边界

- 计划只消费用户目标、可用时间、选择的专项和真实答题记录；不展示预置的完成数据或错题。
- 站外补录必须保留原话、自动提交时间、识别模块和真实用时；可据此调整“当前任务”的推荐顺序并说明原因，但不得把对应周计划任务伪装成完成，也不得凭一句补录推断正确率或掌握程度。
- 补录提交后必须在首页当前任务附近明确显示新增实际用时、当前推荐是否变化和记录入口；没有变化时也要说明原因，不能只用短暂 Toast 表示成功。
- 一句话包含多项学习时，必须分别识别每一项的模块与用时并按合计计入实际学习；不得只取第一个时长再用整句话的最后一个模块覆盖。
- 一句话包含多项学习时，必须分别识别每一项的模块与用时并按合计计入实际学习；不得只取第一个时长再用整句话的最后一个模块覆盖。
- 系统练习的动态调整必须来自真实作答结果：部分完成继续作为当前任务但不算完成；低于目标正确率时优先现有的同类剩余任务并写清原因；没有可调整任务时只进入错题复习，不虚构新的周计划。
- 周计划按任务真实开始和结束时间定位，时间刻度只组织信息，不得为了填满版面伪造任务时段；手机端保留可读卡片宽度并横向查看一周。
- 周计划的模块筛选必须真正隐藏其他模块；完成、部分完成、取消或重排状态不得绕过分类筛选。再次点击当前模块时恢复全部任务。
- 周计划已完成任务必须明显降低饱和度和对比度，同时保留可读文字、完成勾和详情点击；悬停或键盘聚焦时可以略微增强，但不得恢复到与待完成任务同等视觉权重。
- 当前任务卡负责决策和直接开始，任务详情负责日程编辑、重排、题数与真实学习记录；同一段标题、目标和自动记录说明不得在连续两层重复展示。
- “调整计划”应直接进入编辑状态；普通任务的“开始学习”应直接进入练习。只有听写等确有开始前设置的独立模块才保留中间详情。
- 任务显示标题、训练说明和题型编号必须使用同一分类内的索引配对，不能用全周任务序号把写作 51 题显示成图表说明等错误标题。
- 新错题必须保留当前题的题图、原文/音频路径、题干、全部选项、用户选择和正确答案；改进建议要引用当前题的具体关键词与选项差异，不用通用模板。
- 错因只能依据当前题与真实作答行为判断：词汇题可归为“生词影响”，单题可见作答超过3分钟才归为“时间分配问题”；切到其他页面的时间和听力播放等待不得算入超时。
- 单次错题的“复习这道错题”必须直接重做保存的原题，不得偷偷生成一组新题；只有同类错因重复出现时才按重复次数提供 5、10、20 题同型练习。
- 完成错题复习或针对练习后，必须回写本次正确率、最近复习时间、下一次复习日和 1/3/7/14 天阶段；未通过安排次日重做，通过才推进阶段，不能只更新一行练习进度。
- 练习反馈中的“题目要求”只解释当前题在问什么，不复述听力或阅读原文；原文与中文释义只展示一次，再单独给出答题解析。
- 阅读题答题阶段必须把简短题目要求、阅读原文和选项分区展示；不得把整段原文拼进大号题干。提交后再展示原文中文释义，不提前泄露答案线索。
- 在线出题超时或不可用时，兜底题必须正常返回并保持当前考试、等级和任务模块一致；不得让阅读任务混入听力或语法题，任务卡、练习弹窗和剩余题数都必须以实际返回题量为准。
- 5 题练习的在线生成等待不得超过 12 秒；超时后直接使用已验证题库，不为等待少量可能通过质量检查的在线题让用户停留近一分钟。前端等待只可额外预留网络传输缓冲。
- 真题、教材及原始听力音频优先；AI 只做补充练习、解析或无原始音频时的朗读回退，不伪装为真题。
- 真题必须按考试原始题目要求分组并使用严格 `matchTerms` 路由；同属听力或阅读不代表可以混用。每张裁切题图必须完整包含当前题及全部选项，并排除相邻题、页码碎片和缺失选项。
- 听力优先播放 `audioSrc`；对话类回退朗读需保持说话人区分。
- 听写播放入口必须显示准备、播放中、完成和失败状态，不能只保留一个无反馈的文字按钮。
- 听写桌面端使用居中的单任务工作区，标题、进度与练习卡保持同一内容宽度；不得把主练习贴左并在右侧留下失衡的大块空白，也不为填空强塞无关侧栏。
- 听写桌面端使用居中的单任务工作区，标题、进度与练习卡保持同一内容宽度；不得把主练习贴左并在右侧留下失衡的大块空白，也不为填空强塞无关侧栏。
- 听写手写区只有在识别结果能回填答案框时才称为“手写输入”；浏览器缺少韩文手写模型时必须明确回退到系统韩文输入法，不能把纯草稿画布伪装成可判题输入。
- 用户明确同意在线手写识别后，设备本地模型不可用或无结果时可把当前画布图片发送到已配置的视觉模型；图片只用于当次识别，不持久化、不记录原图，并必须显示识别中、失败与候选结果。
- 用户明确同意在线手写识别后，设备本地模型不可用或无结果时可把当前画布图片发送到已配置的视觉模型；图片只用于当次识别，不持久化、不记录原图，并必须显示识别中、失败与候选结果。
- 听写答案卡的整块可见输入区域都必须能聚焦真实韩文输入框；移动端系统手写不能只在内层白色小区域生效。电脑端的独立手写识别区仍通过明确入口展开。
- 听写答案卡的整块可见输入区域都必须能聚焦真实韩文输入框；移动端系统手写不能只在内层白色小区域生效。电脑端的独立手写识别区仍通过明确入口展开。
- 听写核对结果必须直接写“回答正确”或“答案不正确”，并让颜色、正确答案和复习清单状态一致；不得使用“再对照一次”等需要用户猜测的结论。
- 听写核对结果只能在当前词存在非空作答时显示；旧数据出现“已核对但答案为空”时必须按未作答处理，不能一进入页面就把“未输入”判为错误。周计划听写恢复时以当前词已保存的答案为准，不能串用上一词状态。
- 从周计划进入听写时必须绑定真实任务：记录开始、暂停、继续、实际用时、正确数和不熟词；只有整组核对完成才标记计划完成，独立听写模式不得因此被强制计入计划。
- 听写进度中的不熟词数量有数据时必须能直接进入真实不熟词重做；没有数据时明确显示空状态并禁用，不能保留看似可点击的无效卡片。
- 单词消除使用韩文和中文混排卡片墙，允许任一语言先选；正确时只消除同 ID 两张卡，错误时保留，进度只能由真实正确配对增加。
- 听力兜底题的题干、选项、答案、原文与朗读文本必须逐题对应；除明确标注的一组材料多题外，不得用同一段原文批量填充不同题目。
- Reward Engine 是学习完成后的反馈层。除非明确请求，不改 Day1、Day7、RewardScene、PWA 或 Day7 状态机。
- 不把 API Key、环境变量值或私有资料写入仓库文档。
- 当前生产提供方分工固定为：在线出题和手写识别使用智增增承载的 OpenAI 兼容模型（`AI_PROVIDER=openai`、`OPENAI_ALLOW_CUSTOM_BASE_URL=true`），无原始音频时的 TTS 使用已配置的 MiniMax（`TTS_PROVIDER=minimax`）。
- Qwen 和 ElevenLabs 不参与默认链路；不得因检测到其密钥就自动抢占当前提供方。原题 `audioSrc` 始终优先于任何 TTS。
- 历史错题导入只宣称已实现的图片格式；当前为 JPG、PNG、WebP，一次最多3张，PDF未实现前不得显示为可选格式。
- 错题照片必须经过真实视觉识别与逐题确认；看不清的题干、选项和答案保持空缺并提示确认，禁止用固定样例或模型猜测冒充真实原题。
- 导入时保存题干、全部可见选项、用户答案、正确答案、解析和来源快照；首次复习直接重做原题，重复错因才升级为5/10/20题补强。
- 主站视觉统一由 `visual-system.css` 管理：结构界面使用冷静的纸面中性色、深墨蓝骨架和森林绿主操作；课程表模块色仅用于区分真实学习类型。
- 默认表面圆角为 18px、控件圆角为 11px；胶囊形只用于标签和状态。正文与可操作文字不得小于 11px，主要操作不得依靠图标或颜色才能理解。
- 视觉方向固定为中等变化、低动效、中低密度：不使用花哨渐变、满屏卡片、装饰性英文或同级按钮争抢注意力；系统深色模式必须保留真实模块色和可读对比。
- 全站视觉强化优先调整比例、节奏、字重、留白和状态；不得为了“高级感”新增未记录的色彩、渐变、玻璃效果或装饰动效。
- 每页只突出一个主要按钮；次要操作保留，但必须降低视觉权重。禁止用大面积深色卡片、嵌套灰色卡片或装饰性符号制造层级。
- 计划设置只包含会影响计划生成的目标、基础、时间和资料；每日提醒等日常工具使用首页独立入口，不得继续混入计划生成表单。
- 计划设置采用“学习目标 -> 学习时间 -> 补充资料”三步；前两步使用可直接确认的默认值，资料为可选项。重新生成计划时只替换没有真实学习轨迹的任务，完成、部分完成、取消和调整记录必须保留。
- 生成周计划期间必须在主操作附近持续显示当前阶段和进度；完成后返回周计划，失败时保留弹窗、说明记录不会被修改并恢复可重试按钮。
- 不得把同一模块永久绑定到同一钟点；没有真实表现数据时按天轮换所选模块的学习时段，有真实正确率和错题后才给测出的最弱模块增加频次。
- 计划时间在用户可用范围内稳定错峰，按天使用不同的 5 分钟刻度并覆盖完整可用时段；不得每次刷新随机变化，也不得跨越午饭、晚饭或造成任务重叠。
- 不得把同一模块永久绑定到同一钟点；没有真实表现数据时按天轮换所选模块的学习时段，有真实正确率和错题后才给测出的最弱模块增加频次。
- 第一轮计划必须按用户选择的周数生成真实多周任务；每项任务保存 `weekIndex`，周筛选、统计、完成记录与未完成重排均按周隔离。手机打开本周时定位到今天，未来周定位到该周第一项任务；不得用同一周数据冒充翻页。
- 时间线卡片的几何、字号、完成态和悬停态只允许在 `styles.css` 的卡片组件区维护，`visual-system.css` 不得再次覆盖同一选择器。卡片内长注释可以单行省略，但桌面悬停和键盘聚焦必须显示完整标题、注释、时间与状态；手机仍以点击任务详情为完整入口。
- 当前已启用第102届 TOPIK II 听力1-50题、阅读5-12题；49题路由“内容一致”，50题路由“说明方式”，不得因为共用原始讲解而在同一练习中重复堆叠。

## 提交前检查

- 先查看 `git status` 和 `git diff`，不覆盖用户未提交改动。
- 主站修改至少执行 `node --check app.js`、`node --check api/tts.js`（若文件涉及）和 `git diff --check`。
- 涉及练习题时验证来源图/音频、题干、选项、答案、中文释义和错题记录的一致性。
- 涉及听力时用真实题目手动确认：原音频优先、无音频时的回退、播放状态和失败提示。
- 生产交互优先使用隔离的 Playwright CLI 会话实点；不得连接个人浏览器或读取真实账号状态。若测试生成 `.playwright-cli/` 产物，只保留验收结论，不提交临时快照。
- 全量学习闭环回归使用 `specs/full-regression.cli.js`；云同步上游不可用时只验证同域代理、中文错误与本地数据不受影响，不用真实账号反复登录，也不得把外部服务故障伪装成前端已修复。
- 视觉改动至少检查 1440px、820px 和 390px 三档：按钮悬停与焦点、文本换行、弹窗高度、横向溢出和主操作可见性。无法启动浏览器时不得用构建成功冒充视觉验收，也不得在未说明边界时直接部署。
