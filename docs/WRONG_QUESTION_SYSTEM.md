# 错题系统规格

更新时间：2026-06-25

## 1. 定位

错题集是主项目的核心学习闭环之一。

它不只是存放做错的题，而是帮助用户完成：

```text
错题进入系统
→ 归因整理
→ 到期复习
→ 变式练习
→ 判断是否掌握
→ 影响明日重点 / 触发轻反馈
```

## 2. 错题来源

错题可以来自：

- 每日学习任务中的练习结果
- 模拟题或限时训练
- 用户手动导入的历史错题
- 上传的错题照片或 PDF
- 示例错题体验流程

当前原型中已包含「导入旧错题」入口，支持选择错题照片或 PDF。

## 3. 历史错题导入

历史错题导入用于解决用户已经有一批旧错题，但不想手动重新录入的问题。

当前规划 / 原型能力：

- 支持 JPG、PNG、WebP 照片
- 支持 PDF
- 一次最多导入 3 张
- 上传前自动压缩
- 识别不清楚时要求用户确认
- 识别后展示可导入题目列表
- 用户确认后导入错题集
- 导入后自动安排诊断题

当前原型文案方向：

```text
识别到 3 道错题
2道可直接导入，1道需要确认
确认导入错题集
```

正式版可接入图片识别与题目分析能力。

## 4. 错题信息结构

每道错题至少应包含：

- 错题标题
- 所属考试 / 项目
- 所属模块，例如听力、阅读、写作、口语、词汇、语法等
- 错题重点
- 错误原因
- 题目来源
- 是否到期复习
- 当前掌握状态
- 复习记录
- 变式练习记录

## 5. 错题归因

错题归因用于识别用户真正卡住的原因，而不是只记录「做错了」。

归因方向包括：

- 词汇不熟
- 语法结构不熟
- 听力漏听
- 题干误读
- 判断依据不清
- 时间压力导致失误
- 知识点迁移失败
- 题型策略不熟

错题归因会影响后续复习任务和明日重点。

## 6. 复习队列

错题进入错题集后，不应只停留在列表里。

系统需要根据复习节奏安排：

- 当日复盘
- 次日重做
- 3 天后变式练习
- 7 天后延迟检验
- 14 天后再次确认

当前课程表中已经包含「错题复盘」类型任务，例如：

```text
否定表达错题复盘
本周错题二次检验
到期错题复盘
延迟变式检验
```

## 7. 变式练习

错题解决不等于看懂解析。

用户需要完成同类变式题，证明自己真的掌握了判断路径。

当前原型包含：

- AI 生成练习题
- 到期复习变式题
- 错题诊断练习
- 练习完成后自动记录正确率
- 错题结果写入学习打卡或错题集

示例流程：

```text
重做错题
说出判断依据
完成 5 道同类变式题
根据正确率判断是否继续复习
```

## 8. 掌握标准

错题标记为「已掌握」不能只依赖用户主观点击。

当前掌握标准方向：

```text
连续两次正确完成延迟变式题，并能讲明思路。
```

当错题达到掌握标准后：

- 从高频复习队列中移出
- 进入已掌握知识点统计
- 在之后再次答错时，自动恢复到错题复习队列

当前原型中已有「已掌握知识点」入口和「掌握记录」页面。

## 9. 明日重点联动

错题集会影响明日重点。

如果用户当天有未解决错题或练习错误，明日重点应优先安排：

- 重做对应错题
- 复述判断依据
- 完成同类变式题
- 回到对应知识点补弱

示例：

```text
明天先重做「否定表达漏听」错题：说出判断依据，再完成5道同类变式题。
```

## 10. BreakthroughFeedback 联动

错题解决奖励属于 `breakthrough_feedback`，代表「一个具体卡点被拿下了」。

它不使用全屏 RewardScene，而是使用轻反馈 `BreakthroughFeedback`。

触发场景：

- 首次解决错题
- 连续解决 3 道错题
- 一个知识点从错题队列移出
- 延迟变式题连续两次正确

默认文案：

```text
标题：突破卡点
正文：这道错题不是被跳过了，是被你慢慢拿下了。
```

短版：

```text
标题：卡点通过
正文：这次，真的会了。
```

知识点掌握：

```text
标题：掌握确认
正文：这个知识点，已经从错题队列毕业了。
```

## 11. 后续奖励形态探索

错题奖励可以继续探索：

- 高音通过
- 一枚小徽章
- 一张「我会了」小卡
- 一束短暂亮起的紫色追光

但第一期边界明确：错题奖励只做轻反馈，不进入全屏 RewardScene。
