# 产品总览

更新时间：2026-07-01
项目仓库：https://github.com/bzz0309/adaptive-study-planner

## 1. 产品定位

本产品是一个面向多种考试和长期学习目标的学习陪伴系统。

它不是单纯的学习打卡工具，也不是普通课程表生成器，而是一个 Learning Behavior Driven Study System（学习行为驱动系统）：以学习行为为主线，将「考试 / 学习目标配置」「自适应学习规划」「每日任务执行」「错题强化」「完成后的情绪反馈」「成长回顾」分层组织起来。

核心体验可以概括为：

> 用户每天进入系统，是为了完成一段明确的学习行为；情绪反馈只在完成后轻轻接住她。

## 2. 目标用户

核心用户是正在准备某一类考试、语言考试、资格考试，或长期学习项目的学习者。

当前主项目已支持：

- TOPIK
- IELTS
- 其他自定义考试或学习项目

TOPIK 是当前讨论最深入、实现细节最多的重点场景，但不是产品唯一边界。

## 3. 用户特征

- 有明确考试目标，但容易在长期备考中疲惫
- 需要计划感，也需要情绪支持
- 喜欢阶段性记录和仪式感
- 对冷冰冰的任务提醒接受度较低
- 对泰妍、紫色应援、泰蛋、应援棒、演唱会灯海等元素有情感连接

## 4. 核心体验

主体验由四个分层构成：

```text
学习行为（主）
→ 错题强化（学习能力增强）
→ Reward 反馈（情绪反馈层）
→ Cheer Box（成长回顾层）
```

Study Planner 是主产品。用户每天真正使用的是学习计划、任务执行、完成记录、错题复习和明日重点。

Reward Engine 不再作为产品重心，而是学习完成后的情绪反馈层。

连续学习节点使用全屏紫色应援场景；答题全对、错题解决等高光时刻使用轻反馈，不打断主学习流程。

最终产品主线冻结为：

```text
HomeDashboard
→ 学习执行
→ 错题系统（可选强化）
→ Reward Engine（反馈）
→ My Cheer Box（回顾）
```

用户从主站完成学习任务，系统记录学习行为；错题系统只增强学习能力；如果当天触发关键节点，Reward Engine 只作为情绪反馈出现；My Cheer Box 只用于查看成长轨迹，不作为主导航入口。

My Cheer Box v2 当前承担 Day14 / Day30 / Day50 / Day100 的成长内容表达：Day14 是稳定反馈节点，Day30 是习惯确认节点，Day50 是稳定确认点，Day100 是生活化沉淀。它们只作为成长时间轴展示，不进入抽卡、reveal 或新的 reward flow。

## 5. 产品语气

产品语气应当柔和、肯定、有陪伴感。

不应像系统在下达任务，也不应像普通工具在机械反馈。用户完成学习后，反馈应该更像有人轻轻说：

```text
今天也有好好往前走一点。
```

## 6. 情绪目标

- 让用户觉得自己的学习不是孤单推进的
- 让每一次完成都获得柔和、明确的肯定
- 将长期学习拆解为可被纪念的成长节点
- 用应援演唱会的方式强化「被陪伴」和「被看见」的感受

## 8. 当前项目组成

### 8.1 主项目：多考试学习规划工具

仓库根目录中的主应用，负责：

- 学习信息填写
- 多考试计划生成
- 每日打卡
- 明日重点
- 错题集
- 学习资料与 AI 核验
- 账号与云同步
- 学习完成后的反馈触发

详见：[MAIN_APP_REQUIREMENTS.md](./MAIN_APP_REQUIREMENTS.md)

### 8.2 子项目：Reward Companion System

路径：

```text
reward-companion-system/
```

负责紫色应援风格的奖励表现层，包括：

- 全屏 RewardScene
- Day1 / Day7 等连续奖励节点
- HighlightFeedback / BreakthroughFeedback 轻反馈
- PWA 基础能力

当前已完成并冻结的前两个成长节点：

| 节点 | v2 成长语义 | 情绪 | 当前体验 |
|---|---|---|---|
| Day1 | 被看见 | calm | 第一盏应援灯亮起，建立关系和陪伴感 |
| Day7 | 被记录 | excitement | 卡池待机视频 → 抽卡视频 → SSR 正反面小卡 → 收下 |

Day1 和 Day7 都属于 Reward Engine 的成长标记，不是孤立弹窗奖励。Day1 负责让用户感到“第一条记录被看见”，Day7 负责让一周连续学习变成可收藏、可回看的记录。

详见：[REWARD_SYSTEM_SPEC.md](./REWARD_SYSTEM_SPEC.md)

### 8.3 错题系统

错题系统负责从「做错」走到「真正掌握」的学习闭环。

详见：[WRONG_QUESTION_SYSTEM.md](./WRONG_QUESTION_SYSTEM.md)
