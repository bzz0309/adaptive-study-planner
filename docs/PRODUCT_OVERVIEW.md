# 产品总览

更新时间：2026-06-25  
项目仓库：https://github.com/bzz0309/adaptive-study-planner

## 1. 产品定位

本产品是一个面向多种考试和长期学习目标的学习陪伴系统。

它不是单纯的学习打卡工具，也不是普通课程表生成器，而是一个将「考试 / 学习目标配置」「自适应学习规划」「每日陪伴」「错题闭环」「阶段奖励」结合在一起的成长应援系统。

核心体验可以概括为：

> 用户不是在被提醒学习，而是在每一次坚持后，进入一场只为她亮起的紫色应援。

## 2. 目标用户

核心用户是正在准备某一类考试、语言考试、资格考试，或长期学习项目的学习者。

当前主项目已支持：

- TOPIK
- IELTS
- 其他自定义考试或学习项目

TOPIK 是当前讨论最深入、实现细节最多的重点场景，但不是产品唯一边界。

## 3. 用户特征

- 有明确考试目标，但容易在长期备考中疲惫
- 需要计划感，也需要情绪支持
- 喜欢阶段性记录和仪式感
- 对冷冰冰的任务提醒接受度较低
- 对泰妍、紫色应援、泰蛋、应援棒、演唱会灯海等元素有情感连接

## 4. 核心体验

主体验由四个闭环构成：

```text
填写考试 / 学习目标
→ 生成课程表
→ 每日学习与打卡
→ 生成明日重点 / 错题复习 / 奖励反馈
```

奖励不是产品的独立装饰，而是学习闭环后的情绪回应。

连续学习节点使用全屏紫色应援场景；答题全对、错题解决等高光时刻使用轻反馈，不打断主学习流程。

## 5. 产品语气

产品语气应当柔和、肯定、有陪伴感。

不应像系统在下达任务，也不应像普通工具在机械反馈。用户完成学习后，反馈应该更像有人轻轻说：

```text
今天也有好好往前走一点。
```

## 6. 情绪目标

- 让用户觉得自己的学习不是孤单推进的
- 让每一次完成都获得柔和、明确的肯定
- 将长期学习拆解为可被纪念的成长节点
- 用应援演唱会的方式强化「被陪伴」和「被看见」的感受

## 8. 当前项目组成

### 8.1 主项目：多考试学习规划工具

仓库根目录中的主应用，负责：

- 学习信息填写
- 多考试计划生成
- 每日打卡
- 明日重点
- 错题集
- 学习资料与 AI 核验
- 账号与云同步
- 奖励触发判断

详见：[MAIN_APP_REQUIREMENTS.md](./MAIN_APP_REQUIREMENTS.md)

### 8.2 子项目：Reward Companion System

路径：

```text
reward-companion-system/
```

负责紫色应援风格的奖励表现层，包括：

- 全屏 RewardScene
- Day1 / Day7 等连续奖励节点
- HighlightFeedback / BreakthroughFeedback 轻反馈
- PWA 基础能力

详见：[REWARD_SYSTEM_SPEC.md](./REWARD_SYSTEM_SPEC.md)

### 8.3 错题系统

错题系统负责从「做错」走到「真正掌握」的学习闭环。

详见：[WRONG_QUESTION_SYSTEM.md](./WRONG_QUESTION_SYSTEM.md)
