# TOPIK 学习闭环回归计划

## Application Overview

验证用户从真实目标生成周计划，完成或部分完成学习，自动记录结果，产生错题诊断并调整下一步建议的完整闭环。测试使用隔离浏览器数据，不依赖或修改真实用户记录。

## Test Scenarios

### 1. 新用户与计划生成

**Seed:** 独立浏览器会话，清空本站存储后打开生产站。

#### 1.1. honest-empty-state

**Steps:**
1. 首次打开并关闭设置弹窗。
   - expect: 不显示预置完成任务、正确率、错题或实际用时。
   - expect: 页面明确引导生成计划。
2. 重新打开学习设置，只选择听力和阅读。
   - expect: 三步设置可前进、返回和跳过可选资料。
3. 生成计划。
   - expect: 有明确生成状态，等待超过12秒时说明切换题库或稳定排程。
   - expect: 计划只含听力、阅读及有真实依据时允许的巩固任务。
   - expect: 时间不重叠、不跨午饭和晚饭，且不会每天机械使用同一时刻。

### 2. 周计划与任务状态

**Seed:** 隔离注入一周多模块、多状态任务。

#### 2.1. calendar-filter-and-card-state

**Steps:**
1. 点击阅读筛选。
   - expect: 非阅读卡片真正隐藏，已完成听力也不残留。
2. 检查40、45、60分钟卡片。
   - expect: 高度对应真实时长，标题和状态不裁切。
3. 悬停长说明卡片。
   - expect: 显示完整标题、时间、说明与状态。
4. 打开任务、编辑时间并保存。
   - expect: 返回周计划并定位调整后的任务。
5. 标记部分完成、完成、跳过和取消。
   - expect: 状态和真实记录分别保存，取消保留痕迹，跳过可重排但不算完成。

### 3. 系统练习与错题闭环

**Seed:** 隔离计划中包含听力、阅读、写作和词汇语法任务。

#### 3.1. practice-record-diagnose-review

**Steps:**
1. 分别开始听力和阅读5题练习。
   - expect: 生成最长等待不超过16秒并返回实际题量。
   - expect: 阅读原文、题目要求和选项分区；听力优先原始音频且不提前显示原文。
2. 完成部分题后关闭。
   - expect: 保存部分完成、真实用时和剩余题数，不算任务完成。
3. 完成整组并故意答错。
   - expect: 正确率、错题、当前任务和明日重点同步更新。
   - expect: 错题包含原题、选择、正确答案、具体错因和改进建议。
4. 复习单道错题。
   - expect: 直接重做保存原题，不在线生成新题。
5. 重复同类错因。
   - expect: 只在达到阈值后提供5、10、20题强化并回写复习阶段。

### 4. 听写与单词消除

**Seed:** 独立浏览器会话，使用系统内置词表。

#### 4.1. dictation-and-word-elimination

**Steps:**
1. 第一次进入听写。
   - expect: 当前词处于待作答，不显示“未输入”的错误结果。
2. 播放单词并输入韩文。
   - expect: 播放区显示准备、播放中、完成或失败状态；TTS回退使用MiniMax。
3. 打开电脑手写板并识别。
   - expect: 显示识别中、失败或候选结果；成功候选可回填真实输入框。
4. 核对错误和正确答案。
   - expect: 明确显示答案不正确或回答正确，并显示解释、搭配和例句。
5. 刷新页面。
   - expect: 当前词、答案、弱词和任务进度恢复，不串用上一词状态。
6. 进入单词消除并匹配韩中卡片。
   - expect: 仅正确配对后淡出，错误配对不消失，进度可恢复。

### 5. 站外补录与动态建议

**Seed:** 隔离计划含听力、阅读和词汇语法待完成任务。

#### 5.1. external-record-affects-decision

**Steps:**
1. 补录“背了半小时单词，做了20分钟听力”。
   - expect: 分别记录30分钟词汇语法和20分钟听力，合计50分钟。
   - expect: 首页持续显示新增实际用时、记录入口和当前建议是否改变。
   - expect: 不把任何周计划任务伪装为完成，也不推断正确率。
2. 补录缺少模块或缺少用时的句子。
   - expect: 保存原话并明确为什么不能调整计划。

### 6. 进度、提醒与云同步降级

**Seed:** 独立浏览器会话。

#### 6.1. progress-reminder-cloud-fallback

**Steps:**
1. 打开学习进度。
   - expect: 计划、站内实际和站外补录来源清楚，无假准确率。
2. 设置学习提醒并刷新。
   - expect: 时间和开关恢复，通知未授权时有清楚提示。
3. 使用虚构账号尝试云登录。
   - expect: 外部云项目不可用时显示中文可恢复提示，不显示Failed to fetch，不影响本地学习。
4. 在390px和820px视口检查主要页面。
   - expect: 无页面级横向溢出，主要操作可见，输入框可聚焦。
